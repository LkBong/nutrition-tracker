## Features Implemented

### Authentication
- Email/password signup and login via Supabase Auth
- Route protection via `proxy.ts` (Next.js 16 renamed `middleware.ts` to `proxy.ts`)
- Auto-redirect: unauthenticated → `/login`, authenticated on auth pages → `/dashboard`

### Onboarding
- First-time wizard collecting age, weight, height, sex, activity level, and goal
- TDEE calculation using Mifflin-St Jeor BMR formula × activity multiplier
- Macro split: protein 30%, fat 25%, carbs 45% of TDEE

### Food Identification (3 methods)
- **Barcode**: Camera stream auto-detects barcodes using `@zxing/browser` → Open Food Facts API (free, no key)
- **Photo**: Capture button → image sent to Gemini Vision → returns food name + estimated weight + nutrition
- **Text**: Natural language input (e.g. "2 scrambled eggs") → Gemini returns per-item nutrition JSON

### Food Logging
- PortionSelector: adjust grams, pick meal type (breakfast/lunch/dinner/snack)
- Logs stored in Supabase `food_logs` table with RLS
- All nutrition values stored per 100g and scaled at render time

### Dashboard
- Calorie progress bar (actual vs TDEE goal)
- Macro donut chart (protein/fat/carbs breakdown, Recharts PieChart)
- 7-day calorie trend line chart with goal reference line
- Micro-nutrient grid (fiber, sugar, sodium)

### History
- Daily log view with date navigation (prev/next day)
- Grouped by meal type
- Per-entry calorie + protein display
- Delete entries

### Settings
- Edit body profile (age, weight, height, sex, activity, goal)
- See auto-calculated targets
- Manual override for any target (calories, protein, fat, carbs)
- Sign out

### PWA
- `manifest.json` with green 192×512px icons
- Apple meta tags for iOS home screen install
- Service worker via next-pwa
- `--webpack` build mode (required for next-pwa compatibility)

## Database Schema

Three tables in Supabase with Row Level Security:

- **`profiles`** — extends `auth.users`, stores TDEE inputs + goal overrides
- **`foods`** — shared cache of food nutrition data (per 100g), sourced from Open Food Facts or Gemini
- **`food_logs`** — per-user meal entries (food_id, quantity_g, meal_type, logged_at)

Auto-create trigger: `on_auth_user_created` inserts a blank `profiles` row whenever a user signs up.

## Errors Encountered and Solutions

### 1. `CREATE EXTENSION pg_trgm` aborted entire SQL transaction
**Symptom**: Ran `supabase-schema.sql` in Supabase SQL Editor, appeared to succeed, but none of the tables were created.  
**Cause**: Supabase doesn't allow `CREATE EXTENSION` via the SQL Editor — it aborts the transaction, which rolls back everything after it.  
**Solution**: Enable `pg_trgm` via **Supabase Dashboard → Database → Extensions** (UI toggle), then re-run the schema SQL without the `CREATE EXTENSION` line.

---

### 2. Signup returning 500 "database error saving new user"
**Symptom**: Clicking "Create account" showed a Supabase 500 error.  
**Cause**: The `on_auth_user_created` trigger didn't exist because the schema SQL transaction had been aborted (see issue 1). Supabase tried to fire the trigger on signup, but the trigger wasn't there — causing an internal failure.  
**Solution**: Ran this in the SQL Editor to recreate the trigger:
```sql
DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users;
DROP FUNCTION IF EXISTS handle_new_user();

CREATE FUNCTION handle_new_user()
RETURNS TRIGGER LANGUAGE plpgsql SECURITY DEFINER SET search_path = public AS $$
BEGIN
  INSERT INTO public.profiles (id) VALUES (new.id) ON CONFLICT DO NOTHING;
  RETURN new;
END;
$$;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION handle_new_user();
```

### 3. Signup succeeding but not redirecting (silent fail)
**Symptom**: Clicking "Create account" appeared to do nothing — form cleared, no redirect.  
**Cause**: Supabase email confirmation was enabled by default. `signUp()` returned `session: null` (no active session until email confirmed). The proxy saw no session and redirected from `/onboarding` back to `/login`.  
**Solution**: 
- Short term: **Supabase Dashboard → Authentication → Providers → Email → disable "Confirm email"**
- Code fix added: check `data.session` after signUp and show "Check your email" toast if null, rather than attempting a redirect that will fail.

---

### 4. next-pwa incompatible with Next.js 16 Turbopack
**Symptom**: `npm run build` failed with `ERROR: This build is using Turbopack, with a webpack config and no turbopack config`.  
**Cause**: Next.js 16 enables Turbopack by default. `@ducanh2912/next-pwa` injects webpack configuration which conflicts with Turbopack.  
**Solution**: Added `--webpack` flag to both dev and build scripts in `package.json`:
```json
"dev": "next dev --webpack",
"build": "next build --webpack"
```

---

### 5. `middleware.ts` deprecated in Next.js 16
**Symptom**: Build warning: `The "middleware" file convention is deprecated. Please use "proxy" instead.`  
**Cause**: Next.js 16 renamed the middleware convention from `middleware.ts` to `proxy.ts`, and the exported function from `middleware` to `proxy`.  
**Solution**: Renamed the file and updated the export:
```ts
// proxy.ts
export async function proxy(request: NextRequest) { ... }
```

---

### 6. shadcn Select `onValueChange` TypeScript error
**Symptom**: `Type 'string | null' is not assignable to type 'SetStateAction<string>'`  
**Cause**: The new shadcn/ui uses `@base-ui/react` (not `@radix-ui/react-select`). Base UI's Select passes `string | null` to `onValueChange` instead of just `string`.  
**Solution**: Wrap all Select `onValueChange` handlers with a null fallback:
```ts
onValueChange={(v) => setMealType(v ?? "snack")}
```

---

### 7. CalorieNinjas API was down / discontinued
**Symptom**: All food search and photo recognition calls failing.  
**Cause**: CalorieNinjas API went offline permanently.  
**Solution**: Replaced with Google Gemini:
- Text search: Gemini natural language prompt returns nutrition JSON
- Photo recognition: Gemini vision prompt identifies food + estimates nutrition
- Deleted `lib/calorieninjas.ts`, created `lib/gemini.ts`

### 8. Nutritionix also dropped its free tier
**Symptom**: Attempted to use Nutritionix as CalorieNinjas replacement, found they removed their free tier.  
**Solution**: Went fully Gemini for both text and photo. Single API, single key.

---

### 9. Gemini SDK v0.x (`@google/generative-ai`) returning 404 for model names
**Symptom**: `models/gemini-1.5-flash is not found for API version v1beta`  
**Cause**: The old `@google/generative-ai` package defaults to the `v1beta` endpoint, which no longer serves Gemini 1.5 models.  
**Solution**: Replaced with the newer `@google/genai` package:
```bash
npm uninstall @google/generative-ai
npm install @google/genai
```
Updated `lib/gemini.ts` to use `GoogleGenAI` client and `ai.models.generateContent()`.

---

### 10. Gemini quota `limit: 0` on all models
**Symptom**: `Quota exceeded... limit: 0` — even on first request. Usage dashboard showed 0%.  
**Cause**: Google Cloud project didn't have a billing account linked. The free tier quota (1,500 req/day) is locked behind billing being enabled — even though you're not charged within free limits.  
**Status**: Unresolved at end of session.  
**Solution**: Link a billing account at **console.cloud.google.com/billing**.

---

### 11. Hydration mismatch error on signup page
**Symptom**: React hydration error in browser console about mismatched HTML.  
**Cause**: Browser extensions (LanguageTool, a password manager) injecting `data-lt-installed`, `data-sharkid` attributes and `<shark-icon-container>` elements into the DOM before React hydrated.  
**Solution**: Not an app bug. Test in incognito window to confirm. Safe to ignore in development.

## Architecture Decisions Made

| Decision | Rationale |
|---|---|
| PWA over native iOS app | Single codebase, no Apple Developer account ($99/yr), camera works in Safari |
| Supabase over self-hosted Postgres | Free tier, built-in auth, no server to manage |
| Per-100g storage for nutrition | Allows portion scaling at render time without re-fetching |
| Gemini for both text + photo | Reduces API surface to one key; handles natural language natively |
| Open Food Facts for barcodes | Free, no key, global database, excellent coverage |
| Caching food results in Supabase | Avoids re-querying Gemini for the same food twice |
| `@base-ui/react` via shadcn | shadcn v4 switched from Radix to Base UI — type signatures differ |
